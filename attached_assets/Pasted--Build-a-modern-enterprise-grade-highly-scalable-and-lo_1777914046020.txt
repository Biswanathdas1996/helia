## 📄 DOCUMENT INGESTION PIPELINE (CRITICAL)

Build a robust ingestion pipeline with the following stages:

### Step 1: Document Upload

* Support PDF, DOCX, TXT
* Extract raw text

### Step 2: PII Detection & Removal

* Detect sensitive data (email, phone, SSN, etc.)
* Mask/remove PII using NLP models
* Generate PII removal report

### Step 3: Cross-Document Deduplication

* Perform semantic similarity check across knowledge base
* Use embedding similarity threshold (e.g., cosine similarity)
* Remove duplicate or near-duplicate content
* Track removed segments

### Step 4: Admin Review Workflow

* Show:

  * Original content
  * Removed PII sections
  * Removed duplicate sections
* Require admin approval before ingestion
* Allow rejection with feedback

### Step 5: Chunking Strategy

* Intelligent chunking:

  * Token-based chunking (500–800 tokens)
  * Overlap strategy (10–20%)
* Maintain semantic boundaries

### Step 6: Embedding Generation

* Generate embeddings for each chunk
* Store in MongoDB Atlas Vector DB

### Step 7: Metadata Storage

Store metadata:

* Document name
* Upload date
* Document type
* Keywords
* Key phrases
* Tags
* Chunk IDs
* Source references

### Step 8: Ingestion Report

Generate detailed report:

* Total PII removed
* Duplicate content removed
* Final document size
* Number of chunks created
* Embedding stats
